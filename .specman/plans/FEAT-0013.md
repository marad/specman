## AC-1 (modified): Given a TTY and no positional arguments, `specman install` presents an interactive checklist where the user navigates with arrow keys, toggles selection with space, and confirms with enter; install proceeds for the selected agents.

Change: AC-1 previously said only "presents an interactive checklist". The implementation took the easier path — a numbered-toggle prompt where the user types `1`, `2`, `3` to toggle items and presses Enter to confirm. AC-1 now mandates arrow-key navigation + space-to-toggle + enter-to-confirm, ruling out the numbered UX.

Approach: Rewrite `interactiveSelect` in `src/install.ts` to drive a raw-mode TTY checklist instead of `prompt()`. Introduce a small `Key` discriminated union (`up` | `down` | `space` | `enter` | `cancel`) and a `readKey()` injection seam so tests can drive the function with synthetic key sequences without needing a real terminal. The default `readKey` puts `Deno.stdin` into raw mode, reads bytes, parses CSI arrow escape sequences (`\x1b[A` / `\x1b[B`), and maps them to the `Key` enum. The render loop clears and redraws the checklist after each keystroke (cursor up + erase-to-end-of-line per row), highlighting the active row with a `>` marker and rendering checked items as `[x]` / unchecked as `[ ]`. Enter confirms and returns the selected agents (in canonical order); Ctrl+C / Ctrl+D cancels, restores cooked mode, and returns an empty selection (CLI treats that as "nothing to do" — same as it does today when the user unticks everything). Raw mode is always restored in a `finally` block, including on cancel paths.

Files:
- src/install.ts — replace `interactiveSelect` body; add `Key` type and a default raw-mode `readKey()` implementation; keep the function signature shape (callable with injected I/O for tests).
- src/install_test.ts — update the four `interactiveSelect` tests to drive the function with synthetic `Key` sequences (e.g. `["enter"]`, `["down","space","enter"]`) instead of numeric strings.
